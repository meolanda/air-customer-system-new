# วิธีติดตั้งและแก้ปัญหา Google Apps Script

## 🔴 ปัญหาที่พบบ่อย

### 1. หน้าว่างเปล่า (White Screen)

**สาเหตุที่เป็นไปได้:**
- Google Apps Script ยังไม่ได้ Deploy
- Deploy เป็นเวอร์ชันที่ไม่ถูกต้อง
- Permissions ยังไม่ได้ให้
- Tailwind CDN โหลดช้าหรือล้มเหลว

**วิธีแก้:**

#### ขั้นตอนที่ 1: ตรวจสอบการ Deploy
1. เปิด Google Apps Script Editor
2. กดปุ่ม **Deploy** > **New deployment**
3. เลือก type: **Web app**
4. ตั้งค่าต่างๆ:
   - **Description**: ระบบรับงานบริการแอร์
   - **Execute as**: Me (อีเมลของคุณ)
   - **Who has access**: Anyone (ทุกคน) หรือ Anyone with Google account

5. กด **Deploy**
6. **สำคัญ:** คัดลอก URL ที่ได้ เช่น `https://script.google.com/.../exec`

#### ขั้นตอนที่ 2: ให้ Permissions
เมื่อเปิด URL ครั้งแรก Google จะขอ permissions:
1. กด **Continue** (ต่อไป)
2. เลือกบัญชี Google ของคุณ
3. กด **Advanced** (ขั้นสูง)
4. กด **Go to [ชื่อโปรเจกต์] (unsafe)** - **สำคัญ!**
5. กด **Allow** (อนุญาต)

#### ขั้นตอนที่ 3: รัน Setup Function
1. กลับไปที่ Google Apps Script Editor
2. เลือกฟังก์ชัน `setup` จาก dropdown
3. กดปุ่ม **Run**
4. อนุญาตให้สิทธิ์ตามที่ขอ
5. ดูผลลัพธ์ใน **Execution log** จะต้องเห็น "Setup complete!"

### 2. คำเตือน Browser Console

**คำเตือนที่เห็น:**
```
Unrecognized feature: 'ambient-light-sensor'
Unrecognized feature: 'speaker'
Unrecognized feature: 'vibrate'
Unrecognized feature: 'vr'
cdn.tailwindcss.com should not be used in production
```

**คำอธิบาย:**
- คำเตือนเหล่านี้เป็น **ข้อมูลเชิงให้ความรู้** ไม่ใช่ข้อผิดพลาด
- ไม่ส่งผลต่อการทำงานของระบบ
- สามารถเพิกเฉยได้

**วิธีแก้:**
✅ **ไม่ต้องทำอะไร** - ระบบยังทำงานได้ปกติ

---

### 3. Tailwind CDN โหลดช้าหรือล้มเหลว

**อาการ:**
- หน้าเว็บไม่มีสีสัน
- Layout เสีย
- ปุ่มไม่สวยงาม

**วิธีแก้:**

ไฟล์ `index.html` มี **Fallback CSS** อยู่แล้ว ซึ่งจะทำงานแทน Tailwind ถ้า CDN โหลดไม่สำเร็จ

**ถ้ายังมีปัญหา:**
1. ตรวจสอบว่าเข้าถึง Internet ได้
2. ลองรีเฟรชหน้าเว็บ (F5)
3. ลองเปิดใน Browser อื่น
4. ตรวจสอบ Browser Console (F12) ดู error เพิ่มเติม

## 📋 ขั้นตอนการติดตั้งแบบละเอียด

### ขั้นตอนที่ 1: สร้าง Google Sheets
1. ไปที่ [sheets.google.com](https://sheets.google.com)
2. กด **Blank spreadsheet**
3. ตั้งชื่อ sheet: "ระบบรับงานบริการแอร์"

### ขั้นตอนที่ 2: เปิด Apps Script Editor
1. ใน Google Sheets กด **Extensions** > **Apps Script**
2. จะเปิดหน้าต่าง Editor ใหม่

### ขั้นตอนที่ 3: นำเข้าไฟล์โค้ด
1. ลบโค้ดเดิมใน `Code.gs`
2. เปิดไฟล์ `download/google-apps-script/Code.gs`
3. Copy โค้ดทั้งหมด
4. Paste ลงใน `Code.gs`
5. กด **Save** (Ctrl+S) หรือไอคอนดิสก์

### ขั้นตอนที่ 4: นำเข้าไฟล์ HTML
1. ใน Editor กด **+** (Add file)
2. เลือก **HTML**
3. ตั้งชื่อ: `index`
4. เปิดไฟล์ `download/google-apps-script/index.html`
5. Copy โค้ดทั้งหมด
6. Paste ลงใน `index.html`
7. กด **Save**

### ขั้นตอนที่ 5: รัน Setup
1. กลับไปที่ `Code.gs`
2. เลือกฟังก์ชัน `setup` จาก dropdown
3. กดปุ่ม **Run**
4. ตอบ Yes ถ้าขอให้ให้สิทธิ์ (Authorize)
5. ดูใน **Execution log** ว่าเห็น "Setup complete!"

**setup จะทำ:**
- สร้าง Sheet ชื่อ "งานบริการ"
- สร้าง Folder สำหรับเก็บรูปภาพใน Google Drive

### ขั้นตอนที่ 6: Deploy เป็น Web App
1. กดปุ่ม **Deploy** > **New deployment**
2. กดเลือก type: **Web app**
3. กรอกข้อมูล:
   - **Description**: ระบบรับงานบริการแอร์
   - **Execute as**: Me
   - **Who has access**: Anyone (แนะนำ) หรือ Anyone with Google account

4. กด **Deploy**
5. **สำคัญ:** คัดลอก URL เช่น:
   ```
   https://script.google.com/macros/s/XXXXXXXXXX/exec
   ```

### ขั้นตอนที่ 7: เปิดใช้งาน
1. เปิด URL ที่คัดลอกใน Browser
2. Google จะขอ permissions:
   - กด **Continue**
   - เลือกบัญชีของคุณ
   - กด **Advanced**
   - กด **Go to [ชื่อโปรเจกต์] (unsafe)**
   - กด **Allow**

3. ระบบจะแสดงหน้า Login ให้เลือกผู้ใช้

## 🔧 การแก้ไขปัญหาเฉพาะ

### ปัญหา: รูปภาพไม่โหลด

**วิธีแก้:**
1. ตรวจสอบว่า Folder "Aircon Service Images" ถูกสร้างใน Google Drive
2. ถ้าไม่มี: รันฟังก์ชัน `setup` อีกครั้ง
3. ตรวจสอบ permissions ของ Google Drive

### ปัญหา: บันทึกข้อมูลไม่ได้

**วิธีแก้:**
1. ตรวจสอบ Browser Console (F12) ดู error
2. ตรวจสอบว่า Sheet "งานบริการ" มีอยู่จริง
3. รันฟังก์ชัน `setup` อีกครั้ง

### ปัญหา: Deploy ซ้ำแล้วแต่ข้อมูลไม่อัปเดต

**วิธีแก้:**
1. ใหม่ของ Web app เดิม: **Deploy** > **Manage deployments**
2. กดปุ่มไอคอนรูปดินสอ ✏️ (Edit)
3. กด **Deploy** ใหม่
4. คัดลอก URL ใหม่

## 💡 เคล็ดลับ

### การ Debug
1. เปิด Browser Console (F12)
2. ดู tab **Console** สำหรับ error
3. ดู tab **Network** ว่ามี request ที่ล้มเหลวหรือไม่

### การตรวจสอบ Execution Log
1. ใน Apps Script Editor กด **View** > **Execution log**
2. ดูว่าฟังก์ชันไหนล้มเหลว
3. ดู error message ที่แน่นอน

### การ Test
- ทดลองเปิด URL ใน Browser หลายตัว (Chrome, Firefox, Edge)
- ทดลองในทั้ง Desktop และ Mobile
- ทดลองเพิ่ม/แก้ไข/ลบข้อมูล

## 📞 หากยังมีปัญหา

ถ้าทำตามขั้นตอนทั้งหมดแล้วยังไม่ได้:

1. **ตรวจสอบ Browser Console** (F12) ดู error ที่แน่นอน
2. **ตรวจสอบ Execution Log** ใน Apps Script Editor
3. **รีไวส์ทุกอย่าง**:
   - ลบ deployment เดิม
   - สร้าง Google Sheets ใหม่
   - Copy โค้ดใหม่
   - Deploy ใหม่
4. **ตรวจสอบ Internet Connection** และ firewall
