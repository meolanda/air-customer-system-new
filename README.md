# ❄️ ระบบรับงานบริการแอร์

ระบบจัดการงานรับบริการแอร์ที่รวบรวมข้อมูลลูกค้าจากหลายช่องทาง (LINE/โทร/Walk-in/Facebook) พร้อมระบบติดตามสถานะงานและแบ่งงานตามแผนก

## 📋 จุดประสงค์

- ลดการลืมหรืองานตกด้วยระบบสถานะงานที่ชัดเจน
- จัดการงานตามฝ่ายทำงาน เพื่อความชัดเจนในการรับผิดชอบ
- ติดตามประวัติงานทุกครั้งที่เปลี่ยนสถานะ
- เข้าถึงข้อมูลงานได้รวดเร็วด้วยระบบค้นหา
- ใช้งานได้ง่ายบนมือถือ (Responsive Design)

## 👥 โครงสร้างทีม

| ฝ่าย | จำนวน | หน้าที่ |
|------|-------|---------|
| ฝ่ายแอดมิน | 6 คน | รับเรื่องใหม่, จองคิว, ลงปฏิทิน, ดูแลลูกค้า, ส่งใบเสนอราคา, เสร็จงาน |
| ฝ่ายทำใบเสนอราคา | 1 คน | ทำใบเสนอราคาสำหรับลูกค้า |
| ฝ่ายจัดซื้อ | 1 คน | เช็คราคาอะไหล่ + เสนอราคา |

## 🔄 Status Workflow (8 สถานะ)

### สถานะงานและความหมาย

| สถานะ | ไอคอน | ความหมาย | ฝ่ายรับผิดชอบ |
|--------|---------|------------|------------------|
| รับเรื่องใหม่ | 📥 | ลูกค้าแจ้งงานเข้ามาใหม่ | แอดมิน |
| จองคิว / นัดหมาย | 📋 | แอดมินจองคิวให้ลูกค้า | แอดมิน |
| ขอใบเสนอราคา | 💰 | ต้องทำใบเสนอราคาให้ลูกค้า | ฝ่ายทำใบเสนอราคา |
| เช็คอะไหล่ + เสนอราคา | 🔧 | ต้องเช็คราคาอะไหล่ก่อน | ฝ่ายจัดซื้อ |
| ส่งใบเสนอราคาแล้ว | 📨 | ส่งใบเสนอราคาให้ลูกค้าแล้ว | แอดมิน |
| รอลูกค้าตอบกลับ | ⏳ | รอลูกค้าอนุมัติหรือตอบกลับ | แอดมิน |
| เสร็จสิ้น | 🏁 | งานเสร็จสมบูรณ์ | แอดมิน |
| ยกเลิก | ❌ | ลูกค้ายกเลิกงาน | แอดมิน |

### การเปลี่ยนสถานะ (Status Transitions)

**รับเรื่องใหม่ (📥) ได้เปลี่ยนเป็น:**
- จองคิว / นัดหมาย (📋)
- ขอใบเสนอราคา (💰)
- เช็คอะไหล่ + เสนอราคา (🔧)
- ยกเลิก (❌)

**จองคิว / นัดหมาย (📋) ได้เปลี่ยนเป็น:**
- เสร็จสิ้น (🏁)
- ยกเลิก (❌)

**ขอใบเสนอราคา (💰) ได้เปลี่ยนเป็น:**
- ส่งใบเสนอราคาแล้ว (📨)
- ยกเลิก (❌)

**เช็คอะไหล่ + เสนอราคา (🔧) ได้เปลี่ยนเป็น:**
- ส่งใบเสนอราคาแล้ว (📨)
- ยกเลิก (❌)

**ส่งใบเสนอราคาแล้ว (📨) ได้เปลี่ยนเป็น:**
- รอลูกค้าตอบกลับ (⏳)
- ยกเลิก (❌)

**รอลูกค้าตอบกลับ (⏳) ได้เปลี่ยนเป็น:**
- รับเรื่องใหม่ (📥) - เริ่มงานใหม่
- ยกเลิก (❌)

**เสร็จสิ้น (🏁)** - ไม่สามารถเปลี่ยนต่อ

**ยกเลิก (❌)** - ไม่สามารถเปลี่ยนต่อ

### Workflow แบบเต็ม

```
ลูกค้าแจ้งงาน
    ↓
[รับเรื่องใหม่ 📥]
    ├──→ [จองคิว / นัดหมาย 📋] → [เสร็จสิ้น 🏁]
    ├──→ [ขอใบเสนอราคา 💰] → [ส่งใบเสนอราคาแล้ว 📨] → [รอลูกค้าตอบกลับ ⏳] → [รับเรื่องใหม่ 📥]
    └──→ [เช็คอะไหล่ + เสนอราคา 🔧] → [ส่งใบเสนอราคาแล้ว 📨] → [รอลูกค้าตอบกลับ ⏳] → [รับเรื่องใหม่ 📥]
         หรือ → ยกเลิก (ทุกสถานะยกเลิกได้)
```

## 🎯 ฟีเจอร์หลัก

### ✅ รับและบันทึกงาน
- รับงานจาก 5 ช่องทาง: LINE, โทร, Walk-in, Facebook, อื่นๆ
- บันทึกข้อมูล: ชื่อลูกค้า, เบอร์โทร, ที่อยู่, ประเภทงาน, รายละเอียด, ความเร่งด่วน
- อัปโหลดรูปภาพหน้างาน (สูงสุด 10MB)

### ✅ จัดการงานตามฝ่าย
- **แอดมิน**: เห็นงานทุกสถานะ (รับเรื่องใหม่, จองคิว, ส่งใบเสนอราคาแล้ว, รอลูกค้าตอบกลับ, เสร็จสิ้น, ยกเลิก)
- **ฝ่ายทำใบเสนอราคา**: เห็นเฉพาะงาน "ขอใบเสนอราคา"
- **ฝ่ายจัดซื้อ**: เห็นเฉพาะงาน "เช็คอะไหล่ + เสนอราคา"

### ✅ ค้นหาและกรองงาน
- ค้นหา: ชื่อลูกค้า, เบอร์โทร, เลขที่งาน
- กรอง: ทั้งหมด / รอดำเนินการ / เสร็จสิ้น / ตามสถานะ

### ✅ Dashboard
- สถิติ: งานทั้งหมด, งานรอดำเนินการ, งานเสร็จสิ้น
- ตัวบอกสถานะ: Google Sheets / Local Storage

### ✅ ประวัติงาน
- บันทึกทุกครั้งที่เปลี่ยนสถานะ
- แสดงว่าใครเปลี่ยนเมื่อไหร่
- แสดง timeline ของงานแต่ละงาน

### ✅ จัดเก็บข้อมูล
- **Google Sheets** - Database หลัก (sync ข้อมูลระหว่างทีม)
- **Local Storage** - Backup กรณีไม่ได้ configure Google Sheets

## 🚀 การติดตั้ง

### Prerequisites
- Node.js 18+
- Bun หรือ npm หรือ yarn

### Installation

```bash
# Clone repository
git clone <repository-url>
cd aircon-service-master

# Install dependencies
bun install

# Run development server
bun run dev
```

เปิด [http://localhost:3000](http://localhost:3000)

## 📦 การเชื่อม Google Sheets + Drive

### การ Setup แบบง่าย (ใช้ Google Apps Script)

ดูรายละเอียดและคู่มือใน:
- `download/SETUP_GUIDE.md` - คู่มือการ setup Google Apps Script
- `download/google-apps-script/` - โค้ด Google Apps Script ที่ใช้งานได้เลย

### การ Setup แบบ Full API

สร้างไฟล์ `.env.local`:

```env
GOOGLE_SERVICE_ACCOUNT_EMAIL=xxx@xxx.iam.gserviceaccount.com
GOOGLE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"
GOOGLE_SHEETS_ID=your_spreadsheet_id
GOOGLE_DRIVE_FOLDER_ID=your_folder_id
```

ขั้นตอนรายละเอียดใน `download/SETUP_GUIDE.md`

## 🛠️ Tech Stack

- **Frontend**: Next.js 15, React, TypeScript, Tailwind CSS
- **Backend**: Next.js API Routes
- **Database**: Google Sheets / localStorage (fallback)
- **Storage**: Google Drive (รูปภาพ)
- **Auth**: Service Account (Google Cloud)
- **Icons**: Emoji (native)
- **Styling**: Tailwind CSS

## 📝 วิธีการใช้งาน

### 1. เข้าสู่ระบบ
- เลือกผู้ใช้ตามฝ่าย
- แต่ละฝ่ายจะเห็นงานเฉพาะในหน้าที่รับผิดชอบ

### 2. เพิ่มงานใหม่ (เฉพาะแอดมิน)
- กดปุ่ม "+ เพิ่มงาน"
- เลือกช่องทาง
- กรอกข้อมูลลูกค้าและงาน
- อัปโหลดรูปภาพ (ถ้ามี)
- เลือกความเร่งด่วน
- บันทึก

### 3. เปลี่ยนสถานะงาน
- กดปุ่มสถานะถัดไปในการ์ดงาน
- ระบบจะบันทึกประวัติอัตโนมัติ

### 4. แก้ไข/ลบงาน
- กดปุ่ม "✏️ แก้ไข" แก้ไขข้อมูล
- กดปุ่ม "🗑️ ลบ" ลบงาน

### 5. ค้นหางาน
- พิมพ์ชื่อ, เบอร์โทร, หรือเลขที่งาน
- ใช้ dropdown เลือกสถานะที่ต้องการ
