# 📖 User Stories - ระบบรับงานบริการแอร์

## 🎯 โครงสร้าง User Stories

```
As a [ผู้ใช้]
I want to [สิ่งที่ต้องการ]
So that [เหตุผล/ประโยชน์]
```

---

## 👥 ผู้ใช้และบทบาท

### **1. แอดมิน (Admin)**
- **บทบาท:** รับเรื่องลูกค้า, ประสานงาน, สร้างงานใหม่
- **เป้าหมาย:** จัดการงานได้อย่างรวดเร็ว ไม่พลาด
- **ความถี่การใช้:** สูง (ใช้ทุกวัน)

### **2. ฝ่ายทำใบเสนอราคา (Quotation)**
- **บทบาท:** ทำใบเสนอราคา
- **เป้าหมาย:** เห็นงานที่ต้องทำ เปลี่ยนสถานะได้
- **ความถี่การใช้:** ปานกลาง

### **3. ฝ่ายจัดซื้อ (Procurement)**
- **บทบาท:** เช็คราคาอะไหล่
- **เป้าหมาย:** เห็นงานที่ต้องเช็ค เปลี่ยนสถานะได้
- **ความถี่การใช้:** ปานกลาง

---

## 📋 Epic 1: การรับเรื่องและสร้างงาน

### **User Story 1.1: สร้างงานใหม่**

```gherkin
Feature: สร้างงานใหม่

  As a แอดมิน
  I want to สร้างงานใหม่ได้ง่ายๆ
  So that ฉันไม่ต้องจดข้อมูลลูกค้าในกระดาษ

  Scenario: สร้างงานล้างแอร์สำเร็จ
    Given ฉัน login เป็นแอดมิน
    And ฉันอยู่หน้าหลัก
    When ฉันกดปุ่ม "สร้างงานใหม่"
    And ฉันกรอกข้อมูลลูกค้า:
      | ชื่อลูกค้า | เบอร์โทร | ที่อยู่ | ประเภทงาน | รายละเอียด |
      | สมชาย | 0812345678 | บางนา | ล้างแอร์ | ล้างแอร์ 2 ตัว |
    And ฉันกดบันทึก
    Then งานควรถูกสร้างสำเร็จ
    And สถานะควรเป็น "รับเรื่องใหม่"
    And เลขที่เอกสารควรถูกสร้างอัตโนมัติ
    And ฉันควรเห็นงานใหม่ในรายการ

  Scenario: สร้างงานขาดข้อมูลสำคัญ
    Given ฉัน login เป็นแอดมิน
    And ฉันอยู่หน้าสร้างงาน
    When ฉันไม่กรอกชื่อลูกค้า
    And ฉันกดบันทึก
    Then ฉันควรเห็น error "กรุณากรอกชื่อลูกค้า"
    And งานไม่ควรถูกสร้าง

  Scenario: สร้างงานพร้อมอัปโหลดรูป
    Given ฉัน login เป็นแอดมิน
    And ฉันอยู่หน้าสร้างงาน
    When ฉันกรอกข้อมูลครบถ้วน
    And ฉันอัปโหลดรูปภาพปัญหาแอร์
    And ฉันกดบันทึก
    Then งานควรถูกสร้างสำเร็จ
    And รูปภาพควรถูกบันทึกใน Google Drive
    And URL รูปควรถูกเก็บในงาน
```

### **User Story 1.2: เลือกช่องทางติดต่อ**

```gherkin
Feature: เลือกช่องทางติดต่อ

  As a แอดมิน
  I want to ระบุช่องทางที่ลูกค้าติดต่อมา
  So that ฉันรู้ว่าลูกค้ามาจากช่องทางไหน

  Scenario: ระบุช่องทางติดต่อ
    Given ฉันกำลังสร้างงานใหม่
    When ฉันเลือกช่องทาง "LINE"
    And ฉันบันทึกงาน
    Then งานควรมีช่องทางเป็น "LINE"
```

## 📋 Epic 2: การเปลี่ยนสถานะ

### **User Story 2.1: เปลี่ยนสถานะงาน**

```gherkin
Feature: เปลี่ยนสถานะงาน

  As a พนักงาน
  I want to เปลี่ยนสถานะงานได้ง่ายๆ
  So that ฉันรู้ว่างานอยู่ที่สถานะไหน

  Scenario: เปลี่ยนสถานะจาก "รับเรื่องใหม่" เป็น "ขอคิว"
    Given มีงาน "REQ-20260221-001" สถานะ "รับเรื่องใหม่"
    And ฉันเป็นแอดมิน
    When ฉันกดปุ่ม "ขอคิว"
    Then สถานะงานควรเปลี่ยนเป็น "ขอคิว"
    And ประวัติควรบันทึกว่า "เปลี่ยนจาก รับเรื่องใหม่ → ขอคิว"
    And เวลาที่เปลี่ยนควรถูกบันทึก

  Scenario: เปลี่ยนสถานะไม่ได้ (ไม่มีสิทธิ์)
    Given มีงาน "REQ-20260221-001"
    And ฉันเป็นลูกค้า (ไม่ใช่พนักงาน)
    When ฉันพยายาเปลี่ยนสถานะ
    Then ฉันไม่ควรเห็นปุ่มเปลี่ยนสถานะ

  Scenario: เปลี่ยนสถานะตาม workflow ถูกต้อง
    Given งานสถานะ "รับเรื่องใหม่"
    When ฉันเปลี่ยนเป็น "ขอคิว"
    And ฉันเปลี่ยนเป็น "ลงปฏิทินแล้ว"
    And ฉันเปลี่ยนเป็น "ส่งลูกค้าแล้ว"
    Then ทุกครั้งควรบันทึกประวัติ
    And ประวัติควรเรียงตามลำดับเวลา
```

## 📋 Epic 3: การติดตามงาน

### **User Story 3.1: ดูรายการงานทั้งหมด**

```gherkin
Feature: ดูรายการงานทั้งหมด

  As a พนักงาน
  I want to ดูรายการงานทั้งหมดได้แบบสรุป
  So that ฉันรู้ว่ามีงานอะไรอยู่บ้าง

  Scenario: ดู Dashboard สรุป
    Given ฉัน login เข้าสู่ระบบ
    When ฉันอยู่หน้าหลัก
    Then ฉันควรเห็นสถิติ:
      | งานทั้งหมด | งานที่รอดำเนินการ | งานที่เสร็จสิ้น |
      | 15 | 10 | 5 |

  Scenario: ค้นหางานด้วยชื่อลูกค้า
    Given มีงานของลูกค้า "สมชาย" 3 งาน
    When ฉันพิมพ์ "สมชาย" ในช่องค้นหา
    Then ฉันควรเห็นเฉพาะงานของ "สมชาย"
    And งานอื่นควรถูกซ่อน

  Scenario: กรองงานตามสถานะ
    Given มีงานหลายสถานะ
    When ฉันเลือกกรอง "สถานะ: ขอใบเสนอราคา"
    Then ฉันควรเห็นเฉพาะงานสถานะ "ขอใบเสนอราคา"
```

## 📋 Epic 4: การแก้ไขงาน

### **User Story 4.1: แก้ไขข้อมูลงาน**

```gherkin
Feature: แก้ไขข้อมูลงาน

  As a แอดมิน
  I want to แก้ไขข้อมูลงานได้
  So that ฉันสามารถแก้ไขข้อมูลที่ผิดพลาด

  Scenario: แก้ไขชื่อลูกค้า
    Given มีงาน "REQ-20260221-001"
    And ชื่อลูกค้าคือ "สมชาย" (สะกดผิด)
    When ฉันกดปุ่ม "แก้ไข"
    And ฉันแก้ชื่อเป็น "สมชาย"
    And ฉันกดบันทึก
    Then ชื่อลูกค้าควรเปลี่ยนเป็น "สมชาย"
    And ประวัติควรบันทึกการแก้ไข

  Scenario: แก้ไขและเปลี่ยนสถานะพร้อมกัน
    Given มีงาน "REQ-20260221-001" สถานะ "รับเรื่องใหม่"
    When ฉันกดปุ่ม "แก้ไข"
    And ฉันแก้ชื่อลูกค้า
    And ฉันเปลี่ยนสถานะเป็น "ขอคิว"
    And ฉันกดบันทึก
    Then ชื่อลูกค้าควรเปลี่ยน
    And สถานะควรเปลี่ยนเป็น "ขอคิว"
    And ประวัติควรบันทึกทั้ง 2 อย่าง
```

## 📋 Epic 5: การลบงาน

### **User Story 5.1: ลบงานที่ผิดพลาด**

```gherkin
Feature: ลบงานที่ผิดพลาด

  As a แอดมิน
  I want to ลบงานที่สร้างผิดพลาด
  So that ระบบไม่รกจัด

  Scenario: ลบงานที่สร้างผิด
    Given มีงาน "REQ-20260221-999" ที่สร้างผิด
    And ฉันเป็นแอดมิน
    When ฉันกดปุ่ม "ลบ"
    And ฉันยืนยืนยันการลบ
    Then งานควรถูกลบ
    And งานไม่ควรปรากฏในรายการ
    And Google Sheets ไม่ควรมีข้อมูลงานนี้

  Scenario: ลบงานแต่ไม่ยืนยัน (ป้องกันลบผิด)
    Given มีงาน "REQ-20260221-001"
    When ฉันกดปุ่ม "ลบ"
    But ฉันไม่ยืนยัน
    Then งานไม่ควรถูกลบ
    And งานควรยังอยู่ในรายการ
```

## 📋 Epic 6: การดูประวัติ

### **User Story 6.1: ดู Timeline การเปลี่ยนสถานะ**

```gherkin
Feature: ดูประวัติการเปลี่ยนสถานะ

  As a พนักงาน
  I want to ดูประวัติการเปลี่ยนสถานะ
  So that ฉันรู้ว่าใครทำอะไรเมื่อไหร

  Scenario: ดู Timeline การเปลี่ยนสถานะ
    Given มีงาน "REQ-20260221-001"
    And งานนี้มีประวัติ:
      | เวลา | สถานะ | ผู้ทำ |
      | 10:00 | รับเรื่องใหม่ | คุณเนย |
      | 10:15 | ขอคิว | คุณเนย |
      | 14:00 | ลงปฏิทินแล้ว | คุณเนย |
    When ฉันเปิดดูรายละเอียดงาน
    Then ฉันควรเห็น Timeline การเปลี่ยนสถานะ
    And Timeline ควรแสดงเวลา, สถานะ, ผู้ทำ
    And Timeline ควรเรียงตามลำดับเวลา
```

## 📋 Epic 7: การส่งออกข้อมูล

### **User Story 7.1: ส่งออกข้อมูลเป็น Excel**

```gherkin
Feature: ส่งออกข้อมูลเป็น Excel

  As a แอดมิน
  I want to ส่งออกข้อมูลงานเป็น Excel
  So that ฉันสามารถวิเคราะห์ข้อมูลได้

  Scenario: ส่งออกข้อมูลทั้งหมด
    Given มีงาน 15 งานในระบบ
    And ฉันเป็นแอดมิน
    When ฉันกดปุ่ม "ส่งออกข้อมูล"
    And ฉันเลือกรูปแบบ "Excel"
    Then ไฟล์ Excel ควรถูกดาวน์โหลด
    And ไฟล์ควรมีข้อมูลงานทั้งหมด
    And ไฟล์ควรมี header: เลขที่, ชื่อลูกค้า, สถานะ, วันที่

  Scenario: ส่งออกข้อมูลเฉพาะช่วงเวลา
    Given มีงานตั้งแต่วันที่ 1 ถึงวันที่ 28
    And ฉันเป็นแอดมิน
    When ฉันเลือกช่วงเวลา "1 ก.พ. - 28 ก.พ."
    And ฉันกดปุ่ม "ส่งออกข้อมูล"
    Then ไฟล์ Excel ควรมีเฉพาะงานในช่วงเวลาที่เลือก
    And งานนอกช่วงเวลาไม่ควรถูกส่งออก
```

## 🎯 Acceptance Criteria

### **ทุก User Story ต้องมี:**

1. ✅ **Given-When-Then** ชัดเจน
2. ✅ **Acceptance Criteria** ชัดเจน
3. ✅ **Edge Cases** ครอบคลุม
4. ✅ **Testing Steps** ชัดเจน
5. ✅ **Success Metrics** วัดผลได้
